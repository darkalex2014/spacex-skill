# 投标项目全流程管理

## 技能描述

管理投标项目从获取招标文件到交付投标文件的完整生命周期，包括：投标前的招标文件解析与应标判断、投标中的任务分解与进度跟踪、投标后的复盘归档与交接。

---

## 三阶段总览

```
阶段一：投标前（获取招标文件 → 决定是否应标）
阶段二：投标中（任务分解 → 执行 → 提交）
阶段三：投标后（复盘 → 归档 → 交接）
```

---

## 阶段一：投标前

### 第一步：招标文件解析

当用户提供招标文件路径时，立即执行以下分析：

| 分析项 | 输出 |
|-------|------|
| 项目基本信息 | 项目名称、采购单位、预算金额、发布时间、截止时间 |
| 资质门槛 | 营业执照、行业资质、业绩要求、人员证书 |
| 技术要求 | 核心功能、技术指标、验收标准 |
| 评分结构 | 技术标权重、商务标权重、报价权重 |
| 特殊条款 | 围标要求、多公司要求、授权要求 |

**快速提取法**：用 LLM 直接读取招标文件前20页 + 最后一章（评分标准），快速输出摘要。

### 第二步：应标判断

基于招标文件分析结果，判断是否应标：

| 判断维度 | 问题 | 决策依据 |
|---------|------|---------|
| 资质匹配 | 我方是否满足全部资质门槛？ | 不满足 → 不应标 |
| 业务匹配 | 核心需求是否在我方能力范围内？ | 偏离核心 → 不应标 |
| 时间可行 | 剩余时间是否足够完成高质量投标？ | 时间不足 → 不应标 |
| 竞争态势 | 是否有强竞争对手、是否值得投？ | 战略性放弃 → 不应标 |
| 利润空间 | 预算价格是否支撑合理利润？ | 低于成本 → 不应标 |

**结论输出**：
- ✅ **应标**：进入阶段二，同时告知用户预计投入人天
- ❌ **不应标**：告知用户具体原因（如资质不满足/时间不够/业务偏离）

### 第三步：应标决策记录

不应标的项目也需记录：
```markdown
## 不应标记录
- 项目：[名称]
- 采购单位：[单位]
- 不应标原因：[具体原因]
- 日期：[判断日期]
```

## 阶段二：投标中

### 任务分解

应标决策确定后，立即拆解任务：

| 任务类型 | 典型内容 | 执行者 | 截止节点 |
|---------|---------|-------|---------|
| 招标文件精读 | 全文通读，提取技术要求、评分标准 | 顾问/销售 |
| 技术方案编写 | 按评分点逐一编写应答章节 | 顾问/技术 |
| 商务资质准备 | 营业执照、资质证书、业绩合同 | 商务/销售 |
| 报价测算 | 分项报价、总价核算、价格策略 | 财务/销售 |
| 整体校验 | 格式审查、一致性检查、签字盖章 | 顾问/销售 |

### 进度管理

**里程碑设置**：
```
D+N: 完成招标文件精读 → 输出精读摘要
D+N: 完成技术方案初稿 → 输出V1
D+N: 完成技术方案终稿 → 输出V5
D+N: 完成商务材料准备
D+N: 完成报价测算
D-N: 提交前3天 → 整体校验
D-N: 提交前1天 → 最终备份 + 提交
```

**进度跟踪**：每个里程碑完成时汇报给用户，告知当前状态和下一步。

### 多公司模式管理

如果招标文件要求多家主体参与：
1. 识别需要哪些公司（A/B/主公司）
2. 为每家公司分配差异化任务
3. 追踪各公司的进度和质量
4. 汇总前执行串标检测

### 交付前检查清单

| 检查项 | 方法 | 通过标准 |
|-------|------|---------|
| 资质文件完整 | 对比招标文件资质清单 | 全部提供 |
| 技术内容覆盖 | 对照评分点拆解报告 | 每个得分点有内容 |
| 关键参数一致 | 检查人月/工期/性能指标 | 数值一致 |
| 格式规范 | 检查页眉页脚/字体字号 | 符合招标文件要求 |
| 签字盖章 | 检查授权文件、盖章页 | 全部到位 |
| 备份留存 | 打包上传到云盘/共享目录 | 全部存档 |

## 阶段三：投标后

### 开标结果记录

无论是否中标，都记录结果：

```markdown
## 开标结果记录
- 项目：[名称]
- 开标日期：[日期]
- 结果：中标 / 未中标
- 中标单位：[如有]
- 我方报价：[如有]
- 主要差距：[如有，未中标时填写]
- 评审意见：[如有]
```

### 复盘

| 结果 | 复盘重点 |
|------|---------|
| 中标 | 总结成功要素，下次复用 |
| 未中标 | 分析差距（价格/技术/商务/关系），记录教训 |

复盘后更新：
- 业绩库（中标项目加入）
- 知识库（本次招标文件/应答文件归档）
- 纠错库（如有判断失误，更新决策规则）

### 归档

| 归档内容 | 存放位置 |
|---------|---------|
| 招标文件（原始版） | `06标书制作专区/[项目名]/` |
| 应答文件（各版本） | `06标书制作专区/[项目名]/` |
| 评分点拆解报告 | `06标书制作专区/[项目名]/` |
| 开标结果记录 | `06标书制作专区/[项目名]/info/` |
| 复盘记录 | `memory/memory-投标复盘记录.md` |

## 调用模式

### 常规调用
```
用户：有一个[项目名]的招标文件，[路径]
助手：
  1. [调用 SKILL-投标评分点拆解] → 评分点分析
  2. 给出应标判断 + 原因
  3. 如应标 → 分解任务，告知预计人天
  4. 用户确认后 → 进入投标中阶段
```

### 快速调用（已有项目背景）
```
用户：[项目名]现在进展到什么阶段了？
助手：
  1. 读取项目目录状态
  2. 报告当前里程碑进度
  3. 提示下一步任务和截止时间
```

### 复盘调用
```
用户：[项目名]开标结果如何，复盘一下
助手：
  1. 记录开标结果
  2. 分析差距（如未中标）
  3. 输出复盘记录
  4. 更新业绩库和知识库
```

## 注意事项

1. **应标判断是前置关卡**：不应盲目应标浪费资源，判断失误时要记录原因
2. **时间节点严格把控**：投标截止前1天必须完成所有工作并做好备份
3. **多公司模式要隔离**：各公司的文件分开管理，避免混淆
4. **不复盘则白投**：每次投标无论结果都要复盘，知识在于积累
5. **不归档则难复用**：归档后下次同类项目可快速复用内容和经验
