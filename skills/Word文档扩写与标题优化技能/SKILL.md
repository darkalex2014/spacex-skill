# Word文档扩写与标题优化技能

## 技能描述

专业的Word文档处理技能，能够读取Word文档内容，根据每个标题进行深度扩写（扩写至原文五倍以上），同时对标题结构进行梳理、排序和优化（最多保留三级标题）。

## 核心原则

1. **只输出一个最终版本**：扩写内容 + 标题优化，二合一
2. **扩写至五倍以上**：每个章节深度扩写，内容详尽丰富
3. **素材自适应扩写**：根据文档类型和内容特点，灵活选择扩写维度
4. **方法论深度结合**：结合行业领域方法论进行扩充
5. **三级标题结构**：统一精简为最多三级标题

## 触发场景

当用户提到以下关键词时触发本技能：
- 文档扩写、扩写内容、根据标题扩写
- 标题优化、梳理标题、优化文档结构
- 精简标题层级、三级标题
- 重新编制文档、扩充内容、扩写五倍

## 文档类型识别

- **政策文件类**：重在背景、目标、措施、保障
- **技术方案类**：重在需求、技术路线、架构、实施
- **项目管理类**：重在范围、进度、成本、质量、风险
- **项目可行性类**：重在业务、技术、政策、经济、环境可行性
- **人员管理类**：重在组织、职责、激励、发展
- **质量管理类**：重在标准、流程、控制、改进
- **管理制度类**：重在目的、范围、职责、流程、考核
- **项目申报类**：重在背景、目标、内容、预期成效
- **研究报告类**：重在现状、问题、对策、结论

## 扩写方法论（自适应框架）

### 一、扩写维度矩阵（按需组合）

#### 背景维度
- 历史背景和演进历程
- 政策出台的时代背景
- 行业发展的宏观环境
- 现实需求和存在问题

#### 现状维度
- 市场现状和数据统计
- 主要参与主体分析
- 技术发展水平评估
- 存在的问题和困境

#### 目标维度
- 战略目标和定位
- 阶段性目标和里程碑
- 关键指标和量化标准
- 成功标准和验收条件

#### 措施维度
- 具体工作任务
- 实施路径和方法
- 责任主体和时间节点
- 资源保障和投入

#### 技术维度
- 需求分析、技术路线、架构设计
- 详细设计、关键技术、技术指标
- 部署方案

#### 风险维度
- 风险识别和分类
- 风险等级评估
- 防控措施和应急预案
- 风险监控和预警

#### 保障维度
- 组织保障、制度保障
- 资金保障、技术保障
- 人才保障

#### 成效维度
- 预期效果和价值
- 量化指标和评估方法
- 效益分析和投资回报
- 经验总结和推广价值

### 二、项目可行性分析扩写方法论

针对项目类内容扩写，需从**业务、技术、政策、经济、环境、组织**等多维度进行详尽的可行性分析。

#### 2.1 可行性分析框架（基于IS0 21500 / PMBOK）

**核心原则**：
- 系统性：从宏观到微观全面评估
- 客观性：基于数据和事实进行分析
- 可操作性：分析结果可指导决策
- 动态性：考虑未来变化因素

#### 2.2 业务可行性分析

**定义**：评估项目与组织战略目标的契合度，以及业务需求满足程度。

**分析维度**：

1. **战略一致性分析**
   - 与企业战略规划的匹配程度
   - 对核心竞争力提升的贡献
   - 对业务转型支持的力度
   - 战略目标分解与项目目标的映射关系

2. **业务需求分析**
   - 业务痛点识别与量化
   - 用户需求调研与分析
   - 业务流程优化需求
   - 业务能力提升需求
   - 需求优先级评估（MoSCoW法：必须有、应该有、可以有、不需要）

3. **业务价值分析**
   - 直接经济价值（收入增加、成本降低）
   - 间接战略价值（市场地位、品牌影响）
   - 运营效率提升价值
   - 风险规避价值
   - 客户满意度提升价值

4. **业务影响评估**
   - 对现有业务流程的影响
   - 对组织架构的影响
   - 对人员岗位的影响
   - 对上下游协作的影响
   - 变革管理难度评估

5. **业务可持续性分析**
   - 业务模式的可持续性
   - 市场需求的持久性
   - 竞争格局的演变趋势
   - 业务扩展性和演进路径

**评估指标**：
- 战略契合度评分（1-5分）
- 业务需求覆盖率
- 业务价值实现概率
- 业务影响范围
- 业务可持续性指数

#### 2.3 技术可行性分析

**定义**：评估项目所采用技术的成熟度、适配性和实现能力。

**分析维度**：

1. **技术成熟度评估**
   - 技术生命周期阶段（导入期、成长期、成熟期、衰退期）
   - 技术稳定性与可靠性记录
   - 行业应用案例与效果
   - 技术演进趋势预测
   - 技术替代风险评估

2. **技术适配性分析**
   - 与现有技术架构的兼容性
   - 与现有系统的集成难度
   - 技术栈的一致性评估
   - 技术标准遵循程度
   - 接口规范的匹配度

3. **技术实现能力评估**
   - 核心技术能力评估
   - 技术团队能力匹配度
   - 关键技术难点识别
   - 技术风险评估与应对
   - 技术储备和人才培养需求

4. **技术资源分析**
   - 基础设施条件（网络、服务器、存储）
   - 开发工具和平台支持
   - 测试环境和工具
   - 运维监控工具
   - 第三方服务和中间件

5. **技术方案比选**
   - 方案A vs 方案B vs 方案C对比
   - 技术指标对比（性能、可用性、安全性、可扩展性）
   - 成本效益对比
   - 风险对比
   - 推荐方案及理由

#### 2.4 政策可行性分析

**定义**：评估项目与国家政策、法律法规、行业规范的符合程度。

**分析维度**：

1. **国家政策符合性**
   - 与国家战略规划的契合度（如：十四五规划、数字经济战略）
   - 与产业政策的符合程度
   - 与区域发展政策的协调性
   - 政策支持力度和申报可行性
   - 政策变化风险评估

2. **法律法规合规性**
   - 相关法律法规梳理
   - 行业监管要求符合性
   - 数据安全与隐私保护合规（如：《数据安全法》《个人信息保护法》）
   - 知识产权合规审查
   - 合同法务风险评估

3. **行业标准遵循**
   - 国家标准遵循情况
   - 行业标准执行情况
   - 企业标准制定需求
   - 标准兼容性评估
   - 标准化工作规划

4. **监管要求分析**
   - 行业准入要求
   - 资质许可需求
   - 审批备案流程
   - 监管报告要求
   - 监管关系维护策略

5. **政策环境评估**
   - 政策稳定性评估
   - 政策趋势预测
   - 政策风险应对策略
   - 政策红利把握机会

#### 2.5 经济可行性分析

**定义**：评估项目的投资回报和经济效益。

**分析维度**：

1. **投资估算**
   - 建设投资估算（设备购置、软件许可、工程费用）
   - 实施费用估算（咨询费、培训费、迁移费）
   - 运维费用估算（人力成本、软硬件维护、云服务费用）
   - 流动资金估算
   - 不可预见费估算

2. **成本效益分析**
   - 建设成本明细
   - 年度运营成本
   - 成本节约测算（人力节省、效率提升、错误率降低）
   - 收入增加测算
   - 投资回收期计算

3. **财务指标评估**
   - 净现值（NPV）计算
   - 内部收益率（IRR）计算
   - 投资回收期（Payback Period）
   - 投资回报率（ROI）
   - 成本效益比（CBR）

4. **资金筹措方案**
   - 自有资金来源
   - 外部融资渠道
   - 政府补贴和专项资金申请
   - 分期投入计划
   - 资金使用计划

5. **经济敏感性分析**
   - 成本波动影响分析
   - 收益波动影响分析
   - 关键因素敏感性排序

### 三、项目管理类扩写方法论

#### 3.1 项目管理方法论体系

**五大过程组**：
- 启动过程组：项目章程、相关方识别、需求确认
- 规划过程组：范围规划、进度计划、成本预算、质量计划、风险识别
- 执行过程组：团队建设、资源协调、沟通管理、质量保证
- 监控过程组：绩效监控、变更控制、质量控制、风险监控
- 收尾过程组：验收交付、经验总结、文档归档、资源释放

**十大知识领域**：
- 整合管理、范围管理、进度管理
- 成本管理、质量管理、资源管理
- 沟通管理、风险管理、采购管理
- 相关方管理

#### 3.2 范围管理扩写
- **范围定义**：工作分解结构（WBS）设计原则和方法
- **范围确认**：验收标准制定、验收流程设计
- **范围控制**：变更控制流程、变更影响评估

#### 3.3 进度管理扩写
- **活动定义**：活动清单编制、活动属性定义
- **活动排序**：网络图绘制、关键路径法（CPM）
- **工期估算**：专家判断法、类比估算、三点估算
- **进度计划**：甘特图、里程碑图、关键链法（CCM）

#### 3.4 成本管理扩写
- **成本估算**：类比估算、参数模型、自下而上估算
- **成本预算**：基准建立、预算分配、应急储备
- **成本控制**：挣值管理（EVM）、预测技术、偏差分析

#### 3.5 质量管理扩写
- **质量规划**：质量标准制定、质量度量指标
- **质量保证**：过程审计、质量评审、质量体系
- **质量控制**：七种基本质量工具

#### 3.6 风险管理扩写
- **风险识别**：头脑风暴、德尔菲法、SWOT分析
- **风险分析**：定性分析（概率-影响矩阵）、定量分析
- **风险应对**：规避、转移、减轻、接受策略

### 七、扩写执行策略

#### 7.1 内容扩写原则
- **素材为基**：所有扩写基于原始素材延展
- **逻辑强化**：加强论述的逻辑性和说服力
- **实例丰富**：补充具体案例和数据支撑
- **专业深化**：提升内容的专业性和深度
- **方法论结合**：结合相关领域方法论进行扩充

#### 7.2 扩写深度把控
- 核心章节：深度扩写，详细展开
- 次要章节：适度扩写，提炼要点
- 过渡章节：精简表达，衔接自然

#### 7.3 内容质量保证
- 语言正式、严谨
- 专业术语准确
- 数据来源可靠
- 论述有理有据

### 八、标题结构优化

#### 8.1 层级要求
- 最多保留三级标题（X.X.X格式）
- 超过三级的标题进行合理合并或精简

#### 8.2 编号规范
- 一级标题：X.X（如：5.1）
- 二级标题：X.X.X（如：1.1）
- 三级标题：X.X.X.X（如：1.1.1）

#### 8.3 结构优化原则
- 逻辑清晰，层次分明
- 标题之间不重叠、不遗漏
- 符合阅读习惯和专业规范

## 使用流程

1. **接收文档**：用户提供Word文档路径
2. **提取内容**：读取文档，提取所有标题和正文
3. **类型识别**：判断文档类型
4. **自适应扩写**：结合对应领域方法论，深度扩写至五倍以上
5. **标题优化**：精简为三级标题结构
6. **生成文档**：输出扩写+优化的最终版本
7. **输出结果**：保存到源文件相同目录

## 输出规范

### 文件命名
- 输出文件：`<原文件名>_扩写优化版.docx`
- 保存位置：与源文件相同目录

### 内容规范
- 语言正式、严谨
- 专业术语准确
- 论述逻辑清晰
- 数据支撑充分

### 格式规范
- 标题层级清晰（最多三级）
- 段落结构合理
- 字体格式统一

## 注意事项

1. **内容完整**：保留原文所有核心信息，不遗漏关键内容
2. **素材为基**：所有扩写内容必须基于原始文档素材延展
3. **五倍目标**：扩写比例不低于原文的五倍
4. **三级结构**：标题层级最多三级
5. **自适应扩写**：根据文档类型灵活选择扩写维度和对应方法论
6. **可行性分析**：项目类内容需从业务、技术、政策、经济、环境、组织等维度详尽分析
7. **方法论支撑**：以成熟方法论框架（IS0 21500、PMBOK等）进行结构化梳理
8. **只出一版**：只输出最终扩写+优化版本，不产生中间版本
