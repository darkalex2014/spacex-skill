# 串标检测工程化封装

## 功能说明
基于企业基本信息（工商注册号、统一社会信用代码、企业名称等），通过多维度数据比对，识别投标文件中是否存在围标、串标行为。用于：投标文件合规性审查、招标前风险筛查、采购合规检查。

## 核心能力

- **工商信息交叉核验**：比对不同投标企业的工商登记信息（法定代表人、注册地址、注册资本来源等）
- **关联关系识别**：识别投标企业之间的股权关联、董事交叉任职、历史关系
- **IP地址/设备指纹分析**：比对投标文件提交的IP地址、MAC地址、设备指纹等电子痕迹
- **文件相似度检测**：比对投标文件的电子文档特征（作者、创建时间、修改痕迹、格式指纹）
- **报价异常分析**：识别投标报价的高度规律性（抱团报价、阶梯报价）

## 数据来源

- 企业工商基本信息（注册号、信用代码、法定代表人、注册资本）
- 企业股权穿透数据（股东结构、关联企业）
- 投标文件元数据（电子文档属性）
- 报价历史数据库

## 输出格式

```markdown
串标风险评估报告
├── 基础信息：企业名称、招标项目、投标时间
├── 风险评分：0-100分（综合风险指数）
├── 风险项明细：
│   ├── 关联关系风险：关联企业同时投标
│   ├── 电子痕迹风险：IP地址/MAC地址重复
│   ├── 文件指纹风险：文档特征高度相似
│   └── 报价异常风险：报价规律性异常
└── 结论与建议：高/中/低风险，建议进一步核查
```

## 调用方式

用户：检测串标风险 / 围标审查 / 投标合规检查 / 串标分析
助手：
  1. 接收用户提供的投标企业信息和招标文件
  2. 调用各维度数据接口进行交叉比对
  3. 综合评分并生成风险报告
  4. 输出结构化风险评估结果

## 注意事项

1. **数据范围**：检测结果依赖已有数据覆盖范围，数据缺失可能影响准确率
2. **辅助决策**：本工具提供风险提示，不构成法律合规结论
3. **证据链**：检测到的风险项需结合原始数据作为证据支撑
