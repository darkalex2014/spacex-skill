# 差异化扩写 Prompt 模板

## 技能描述

为多公司投标模式提供可直接使用的差异化扩写 Prompt 模板。将"技术应答文件批量编制技能"中的 B-1/B-2/B-3/B-4 四种风格具象化为可喂给 LLM 的标准 Prompt，支持公司 A / 公司 B / 主公司的差异化生成。

**前置依赖**：已检测出雷同段落，并按相似度分级（100% / 90-99% / 70-89%）。

---

## 扩写维度（五倍扩写框架）

对每个雷同段落，从以下 6 个维度中选择适用的 2-4 个进行扩写：

| 维度 | 扩写方向 | 适用风格 |
|------|---------|---------|
| **背景** | 历史背景、行业环境、政策驱动、现实需求 | B-2/B-4 |
| **现状** | 市场现状、数据统计、主体分析、问题描述 | B-2/B-4 |
| **目标** | 战略目标、量化指标、验收条件、价值阐述 | B-1/B-2 |
| **措施** | 具体任务、实施路径、责任主体、质量要求 | B-1/B-3 |
| **技术** | 技术路线、架构设计、部署方案、选型依据 | B-1/B-2 |
| **保障** | 组织保障、制度保障、人才保障、应急机制 | B-1/B-3 |

---

## 风格定义（输入 Prompt 时的风格指令）

### B-1 · 传统严谨型

**特征**：长复合句、完整论证、层层递进、书面语为主
**适用公司**：主公司（基准版）

**风格指令片段**：
```
你是一位专业的投标方案撰写专家。请采用传统严谨的文风撰写以下内容：
- 使用完整的长句结构，避免短句堆砌
- 论证逻辑：现状描述 → 问题分析 → 方案设计 → 预期效果
- 适当使用"因此""与此同时""基于上述分析"等连接词
- 保持客观专业的语气，避免口语化表达
```

### B-2 · 深度展开型

**特征**：中长段落、多角度论述、细节丰富、有案例支撑
**适用公司**：公司 A（差异化版）

**风格指令片段**：
```
你是一位投标方案撰写专家。请采用深度展开的文风撰写以下内容：
- 每个要点展开为150-300字，提供充分的论证细节
- 从多个角度分析（技术/业务/管理/风险）
- 结合行业最佳实践举例说明
- 可引用相关行业标准、技术规范或成功案例
- 使用"具体而言""以本项目为例""在XX方面"等引出细节
```

### B-3 · 简洁概括型

**特征**：短句分点、直击要点、重点突出、便于快速阅读
**适用公司**：公司 B（高度差异化版）

**风格指令片段**：
```
你是一位投标方案撰写专家。请采用简洁概括的文风撰写以下内容：
- 使用短句，每段不超过3-4句
- 以要点罗列为主，使用"1. 2. 3."或"第一、第二、第三"结构
- 每个要点一句话说清核心，不展开论证
- 删除所有修饰性废话，直奔结论
- 适合时间紧迫或评委快速扫读的场合
```

### B-4 · 案例实证型

**特征**：引用案例、数据支撑、实证分析、可信度高
**适用公司**：公司 B 进阶版（强调差异化时使用）

**风格指令片段**：
```
你是一位投标方案撰写专家。请采用案例实证的文风撰写以下内容：
- 每个论点都需要案例或数据支撑
- 引用类似项目经验时注明项目名称（可用"[XX行业标杆项目]""[国内某大型企业]"等占位符）
- 数据需要具体：量化指标、百分比、时间节点
- 对比分析：我方方案 vs 常规方案，突出优势
- 可使用"根据Gartner/IDC报告""行业调研显示"等第三方背书
```

## 相似度分级 → 扩写倍数 → 维度组合

| 雷同程度 | 扩写倍数 | 目标字数增幅 | 推荐维度组合 | 推荐风格 |
|---------|---------|------------|------------|---------|
| 100% 完全雷同 | **5倍以上** | 100字→500字 | 背景+现状+目标+措施+技术 | B-1 或 B-2 |
| 90-99% 高度雷同 | **4倍** | 100字→400字 | 现状+措施+技术+保障 | B-2 或 B-3 |
| 70-89% 中度雷同 | **2倍** | 100字→200字 | 措施+技术（选2个） | B-3 或 B-4 |

## 扩写 Prompt 模板库

### 模板 A：100%雷同 → B-1 传统严谨（扩写5倍）

```
## 任务
以下段落与围标文件高度雷同，需要深度扩写5倍以上。请基于原文语义进行扩充。

## 原文
[雷同段落文本]

## 扩写要求
1. 风格：传统严谨（B-1），长复合句，层层递进
2. 从以下维度扩充（选择4-5个）：
   - 背景：增加历史背景、行业环境、政策驱动
   - 现状：增加市场现状、主体分析、问题描述
   - 目标：增加战略目标、量化指标、验收条件
   - 措施：增加具体任务、实施路径、责任主体
   - 技术：增加技术路线、架构设计、选型依据
   - 保障：增加组织保障、制度保障、人才保障
3. 保持原文核心语义不变，不改变关键参数（人月/工期/百分比）
4. 输出：扩写后文本（中文，500字以上）

## 格式
直接输出扩写后文本，不需要解释。
```

### 模板 B：100%雷同 → B-2 深度展开（扩写5倍）

```
## 任务
以下段落与围标文件高度雷同，需要深度扩写5倍以上。请基于原文语义进行扩充。

## 原文
[雷同段落文本]

## 扩写要求
1. 风格：深度展开（B-2），多角度论述，细节丰富
2. 从以下维度扩充：
   - 每个要点展开150-300字
   - 从技术、业务、管理、风险至少3个角度分析
   - 结合"[行业标杆案例]"举例说明
   - 使用"具体而言""以本项目为例""在XX方面"引出细节
3. 保持原文核心语义不变，不改变关键参数
4. 输出：扩写后文本（中文，500字以上）
```

### 模板 C：90-99%雷同 → B-3 简洁概括（扩写4倍）

```
## 任务
以下段落与围标文件高度雷同，需要扩写4倍。请基于原文语义进行扩充。

## 原文
[雷同段落文本]

## 扩写要求
1. 风格：简洁概括（B-3），短句分点，直击要点
2. 从以下维度扩充（选择2-3个）：
   - 措施：增加具体任务、实施路径、责任主体
   - 技术：增加技术路线、选型依据
   - 保障：增加组织保障、人才保障
3. 以分点罗列为主，每个要点一句话
4. 删除所有修饰性废话
5. 保持原文核心语义不变，不改变关键参数
6. 输出：扩写后文本（中文，300字以上）
```

### 模板 D：70-89%雷同 → B-4 案例实证（扩写2倍）

```
## 任务
以下段落与围标文件有一定相似度，需要适度扩写2倍。请基于原文语义进行扩充。

## 原文
[雷同段落文本]

## 扩写要求
1. 风格：案例实证（B-4），引用案例、数据支撑
2. 从以下维度扩充（选择1-2个）：
   - 措施：增加具体任务描述
   - 技术：增加技术路线说明
3. 引用类似项目经验（使用"[XX行业案例]""[国内某大型企业]"等占位符）
4. 加入量化数据（如响应时间/并发数/成功率等）
5. 对比分析：常规做法 vs 我方做法
6. 保持原文核心语义不变，不改变关键参数
7. 输出：扩写后文本（中文，200字以上）
```

### 模板 E：全流程扩写（多公司批量处理）

```
## 上下文
- 主公司V5.docx 的 [章节名] 段落已通过串标检测，发现与 [公司A/公司B] 文件存在 [相似度]% 雷同
- 需要为 [公司A/公司B] 生成差异化扩写版本

## 执行步骤
1. 读取 [雷同段落文本]
2. 根据相似度级别选择模板（A/B/C/D）
3. 根据目标公司选择风格（B-1/B-2/B-3/B-4）
4. 生成扩写后文本
5. 自检：对比扩写后文本与原文，确认语义一致但表述完全不同

## 关键参数（禁止修改）
- 人月：XXX人·月
- 工期：XXX个月
- 性能指标：[具体数值]
- 服务承诺：[具体内容]

## 输出格式
```markdown
## 公司[A/B] 扩写版本 - [章节名]

[扩写后文本]
```
```

## 扩写自检清单

扩写完成后，用以下清单检查：

| 检查项 | 通过标准 |
|-------|---------|
| 语义一致性 | 与原文核心意思一致，未改变业务含义 |
| 参数一致性 | 人月/工期/性能指标等数值完全一致 |
| 字数增幅 | 达到对应雷同等级的扩写倍数要求 |
| 风格差异 | 与主公司版本在表述方式上有明显区别 |
| 无新增雷同 | 与其他公司文件对比，相似度降至70%以下 |
| 可读性 | 语句通顺，无语法错误，无乱码 |

## 常见扩写示例

### 示例1：100%雷同段 → B-2深度展开

**原文**（雷同）：
> 项目经理须为正式员工，具备3年以上项目管理经验。

**扩写后（B-2）**：
> 关于项目经理选配，我公司制定了严格的人员准入标准。具体而言，项目经理须为本公司正式员工（须签订正式劳动合同并缴纳社保，不接受劳务派遣或外包形式），且应具备以下条件：①持有PMP（项目管理专业人士）资格认证证书，注册有效；②具备3年以上同类大型信息化项目管理经验，近5年内担任过不少于5个同等规模项目的项目经理职务；③熟悉项目所在行业业务流程，具备跨部门协调和资源整合能力；④近3年内负责的项目未发生重大质量安全事故或严重延期情形。以本项目为例，我公司将安排具有石油石化行业信息化建设经验的项目经理全程跟进，确保项目顺利推进。

### 示例2：90-99%雷同段 → B-3简洁概括

**原文**（雷同）：
> 技术方案采用微服务架构，支持高可用部署。

**扩写后（B-3）**：
> 技术方案架构设计要点：
> 1. 架构选型：采用微服务架构，容器化部署，支持弹性扩缩容
> 2. 高可用方案：多副本部署 + 负载均衡 + 自动故障切换，可用性≥99.9%
> 3. 技术栈：Spring Cloud + Kubernetes + Redis + MySQL
> 4. 部署模式：支持本地私有化部署及云端部署两种模式
